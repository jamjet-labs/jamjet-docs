# 클라우드 vs 오픈소스

JamJet은 함께 작동하는 두 가지 제품입니다. 이 페이지는 필요한 제품을 선택하는 데 도움을 드립니다.

## JamJet Cloud란 무엇인가요?

JamJet Cloud는 AI 에이전트를 위한 **호스팅 거버넌스 플랫폼**입니다. 작은 SDK(`@jamjet/cloud` 또는 `jamjet`)를 설치하고 기존 OpenAI / Anthropic 클라이언트를 래핑하면 다음을 얻을 수 있습니다:

- **정책** — 런타임에 위험한 도구 차단
- **예산** — 프로젝트별 비용 상한선, 호출 전 강제 적용
- **승인** — 민감한 작업에 대한 사람 개입 게이트
- **텔레메트리** — 모든 LLM 호출을 스팬으로 캡처; 대시보드, 검색, 감사
- **멀티 에이전트 추적** — 어떤 에이전트가 무엇을 했는지 보여주는 네트워크 그래프 뷰

앱 구조를 재편할 필요가 없습니다. SDK는 프레임워크가 아니라 래퍼입니다.

## 오픈소스 런타임이란 무엇인가요?

JamJet 오픈소스 런타임은 AI 에이전트를 위한 **내구성 있는 워크플로 엔진**입니다. 핵심은 Rust로 작성되었으며, 작성 인터페이스는 Python과 Java입니다. 다음과 같은 워크플로를 작성할 수 있습니다:

- 모든 노드 이후 상태를 체크포인트하여 프로세스 재시작 시 원활하게 재개
- 결정론적으로 재생하여 버그 재현 가능
- 에이전트 간 통신을 위해 MCP 및 A2A 프로토콜을 네이티브로 지원
- 최소 한 번 전달 보장으로 수평 확장

런타임은 다단계 워크플로를 *실행*해야 할 때 — "LLM 호출 래핑"만으로는 충분하지 않을 때 사용합니다.

## 의사결정 매트릭스

| 원하는 작업 | 사용 제품 |
|---|---|
| 기존 AI 앱(챗봇, 에이전트 루프) 거버넌스 — 도구 차단, 비용 상한선 설정, 대시보드 확보 | **Cloud** |
| 체크포인트 재생, A2A, MCP를 지원하는 다단계 내구성 워크플로 구축 | **오픈소스 런타임** |
| 내구성 있는 워크플로 실행과 거버넌스를 모두 수행 | **둘 다** — Cloud의 SDK가 런타임의 LLM 호출을 자동으로 래핑 |
| 둘 중 하나에 메모리(장기 지식) 추가 | **Engram** (별도 제품, [java-ai-memory.dev](https://java-ai-memory.dev) 참조) |

## 함께 사용하는 방법

Cloud와 오픈소스 런타임은 독립적입니다. 둘 중 하나만 사용해도 되고 둘 다 사용하지 않아도 됩니다. 두 가지를 함께 사용하면 Cloud의 SDK가 런타임의 LLM 호출을 감지하고 이에 대한 스팬을 생성하므로, 추가 연결 작업 없이 런타임 워크플로우에 대한 거버넌스를 확보할 수 있습니다.

## 가격

- **JamJet Cloud:** 넉넉한 한도의 무료 티어 제공. 프로덕션 사용 시 스팬 100만 개당 $0.50부터 시작. [app.jamjet.dev/pricing](https://app.jamjet.dev/pricing) 참조.
- **오픈소스 런타임:** Apache 2.0. 어디서든 실행 가능.
- **Engram:** Apache 2.0 (오픈소스) + Cloud의 호스팅 버전 (베타).
