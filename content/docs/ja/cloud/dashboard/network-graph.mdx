# ネットワークグラフ

**app.jamjet.dev/projects/&lt;id&gt;/graph** のネットワークグラフは、システム内のエージェント同士の関係性をリアルタイムで可視化するキャンバスです。ログでは答えられない疑問に答えます：どのエージェントがどのエージェントに処理を引き渡したのか、リクエストが何ホップを経由したのか、チェーン内のどこでポリシーがツール呼び出しをブロックしたのか、どのエージェントが最も多く接続されているのか。

## グラフの読み方

キャンバス上の各ノードは名前付きエージェントを表します — [`agent()`](/cloud/governance/agents) で宣言する個別のアイデンティティです。各エッジはインタラクションを表します：あるエージェントから別のエージェントへのツール呼び出し、あるエージェントが別のエージェントのスコープ内で実行中に発行されたスパン、またはマルチエージェントワークフローでの明示的な引き渡しです。

ノードのサイズはスパン量に比例します。太いエッジは、選択した時間ウィンドウ内でその2つのエージェント間により多くのインタラクションがあったことを意味します。出力エッジのないノードはリーフエージェントです — コンテキストを消費しますが、さらに委譲することはありません。

ノードをクリックすると**エージェント詳細パネル**が開きます：現在の時間ウィンドウでそのエージェントが発行したすべてのスパンの時系列リストで、それぞれのコスト、モデル、レイテンシ、ポリシー結果が表示されます。エッジをクリックすると**エッジ詳細パネル**が開きます：その特定のエージェント間境界を越えたスパンのサブセットが時間順に表示されます。

## 時間範囲のスコーピング

グラフ右上の時間範囲ピッカーで、どのスパンを含めるかを制御します。利用可能なウィンドウ：

- **過去1時間** — デフォルト。アクティブな開発中やインシデント対応時に有用です。
- **過去24時間** — 1日のトラフィックの広域ビュー。
- **過去7日間** — アーキテクチャレビューや長期にわたるインシデント分析。
- **カスタム範囲** — 任意の開始時刻と終了時刻を選択。グラフはそのウィンドウで再計算されます。

時間範囲を絞り込むことは、インシデントを特定する最も効果的な方法です：異常な動作が発生した正確な間隔にウィンドウを設定し、どのエージェントがアクティブで、どのエッジが活発だったかを検証します。

## フィルター

キャンバスの上部には3つのフィルターコントロールがあります：

- **エージェントフィルター** — 選択したエージェントとその直接の隣接ノードのみを表示します。多数のエージェントがある場合に、特定のサービス境界に集中したいときに便利です。
- **ユーザーフィルター** — 特定のエンドユーザーに提供されたコールチェーンのみを表示します。SDKによって[ユーザーコンテキスト](/cloud/telemetry/user-context)が設定されている必要があります。ユーザーフィルターが有効な場合、そのユーザーに帰属するスパンによって触れられていないエッジとノードはグレーアウトされます。
- **環境フィルター** — [プロセスコンテキスト](/cloud/telemetry/process-context)を介して`environment`タグをスパンに付けている場合、グラフを`production`、`staging`、またはその他の使用している値に制限できます。

フィルターは組み合わせ可能です：過去24時間にわたって`production`環境のユーザー`u_123`を同時に表示できます。

## コーディネータースコアリング注釈

コールチェーンにJamJetオープンソースの[コーディネーターノード](/open-source/research/coordinator)が含まれている場合、ダッシュボードはそれを通過するエッジに追加の注釈をレンダリングします。これらの注釈は、コーディネーターがどの下流エージェントに委譲するかを選択する際に計算したスコアリングデルタ（上位にランク付けされたエージェントと次の代替案との間の数値差）を示します。

デルタが大きい場合、コーディネーターはそのルーティング決定に高い確信度を持っていたことを示します。デルタが小さい場合、僅差だったことを意味し、下流エージェントのパフォーマンスが低下した場合は確認する価値があります。注釈は、コーディネーターが構造化されたスコアリングスパンを出力する場合にのみ表示されます。純粋なヒューリスティックルーティングでは生成されません。

## ユースケース

**インシデントポストモーテム。** 時間範囲をインシデントウィンドウに設定し、本番環境の環境フィルターを適用します。グラフは、どのエージェントが関与し、どのエッジが最も多くのトラフィックを処理したかを即座に明らかにします。高負荷のエッジをクリックして、レイテンシが急増した正確なスパンや、ポリシーがツール呼び出しをブロックした箇所を見つけます。

**アーキテクチャレビュー。** 本番環境で7日間のウィンドウに切り替えます。グラフは、設計したものではなく、実際に出現したエージェントトポロジを明らかにします。予期せず高い接続性を持つノード、エージェント間のサイクル、アクティブなサブオーケストレーターであるべきリーフエージェントなどがすべて可視化されます。

**エージェント間依存関係のトレース。** システムに新しいエージェントを追加する際、最初の24時間にわたってグラフを監視し、意図したエージェントに接続され、接続すべきでないエージェントに予期せず到達していないことを確認します。

**ユーザー別サポートチケット。** 報告ユーザーのIDにユーザーフィルターを適用します。グラフは、そのユーザーのリクエストに提供されたコールチェーンのみを表示するため、何が起こったのか、どの順序で起こったのか、どこで問題が発生したのかを追跡することが容易になります。
