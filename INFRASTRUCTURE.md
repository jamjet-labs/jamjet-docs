# JamJet Docs — Infrastructure & Setup

> How the docs site is built, deployed, and translated.
> Read this if something breaks.

## Architecture

```
content/docs/*.mdx (English source)
        │
        ├──► Crowdin (auto-translates to ja, zh, ko, es, de)
        │       └──► PR with content/docs/{locale}/*.mdx
        │
        ▼
  Fumadocs (Next.js) builds all locales
        │
        ▼
  Static export (out/)
        │
        ▼
  Cloudflare Pages serves docs.jamjet.dev
```

## Stack

| Component | Technology | Purpose |
|-----------|-----------|---------|
| Framework | Fumadocs (Next.js 16) | Docs site with sidebar, search, dark mode |
| Styling | Tailwind CSS v4 | Utility CSS |
| Fonts | Source Serif 4 + Plus Jakarta Sans + IBM Plex Mono | Editorial Engineering theme |
| i18n routing | Fumadocs built-in | `/en/docs/...`, `/ja/docs/...` |
| Translation | Crowdin (free OSS) | Auto-translates MDX on push |
| Hosting | Cloudflare Pages | Free, unlimited bandwidth, global CDN |
| CI/CD | GitHub Actions | Build + deploy on push to main |

## For contributors

**You only need to edit English files.** Write or edit MDX files in `content/docs/`. Crowdin handles translation automatically.

```bash
# Local development
npm install
npm run dev        # http://localhost:3000

# Add a new doc page
# 1. Create content/docs/my-page.mdx with frontmatter:
#    ---
#    title: My Page
#    description: What this page covers.
#    ---
#
# 2. Add "my-page" to content/docs/meta.json in the desired position
# 3. Commit and push — Crowdin translates, Cloudflare deploys
```

### File structure

```
content/docs/
├── meta.json              # Sidebar order and section headings
├── index.mdx              # Landing page (/docs)
├── quickstart.mdx          # Quickstart guide
├── concepts.mdx            # Architecture overview

    └── ...
```

## Deployment

### Cloudflare Pages setup (one-time)

1. Go to [Cloudflare Pages](https://dash.cloudflare.com/?to=/:account/pages)
2. Create project → Connect to `jamjet-labs/jamjet-docs`
3. Build settings:
   - Framework preset: `Next.js (Static HTML Export)`
   - Build command: `npm run build`
   - Output directory: `out`
4. Add custom domain: `docs.jamjet.dev`
5. Add repo secrets in GitHub:
   - `CLOUDFLARE_API_TOKEN` — create at Cloudflare dashboard → API Tokens
   - `CLOUDFLARE_ACCOUNT_ID` — visible in Cloudflare dashboard URL

### DNS (one-time)

Add CNAME record in your DNS provider:
```
docs.jamjet.dev  CNAME  jamjet-docs.pages.dev
```

## Translation (Crowdin)

### Setup (one-time)

1. Create project at [crowdin.com](https://crowdin.com) (free for OSS)
2. Target languages: Japanese, Chinese Simplified, Korean, Spanish, German
3. Install [Crowdin GitHub Integration](https://crowdin.com/integrations/github)
4. Connect to `jamjet-labs/jamjet-docs` repo
5. Enable machine translation pre-translation (Settings → Machine Translation)

### How it works

1. You push English MDX to `content/docs/`
2. Crowdin detects changes, runs AI translation
3. Crowdin opens a PR with translated files at `content/docs/{locale}/`
4. Auto-merge or review, then merge
5. Cloudflare Pages rebuilds with all locales

### Adding a new language

1. Add locale to `src/lib/i18n.ts`:
   ```typescript
   { locale: 'fr', name: 'French' },
   ```
2. Add language in `src/lib/source.ts` `languages` array
3. Add language in Crowdin project settings
4. Crowdin starts translating automatically

## Troubleshooting

### Build fails

```bash
npm run build    # Check error output
```

Common issues:
- **YAML parse error in frontmatter**: Check MDX frontmatter for indentation issues
- **Component not defined**: MDX uses a Starlight component (`<Aside>`, `<Steps>`) — convert to plain markdown
- **Style string in JSX**: Use `style={{prop: "value"}}` not `style="prop: value"`

### Translations not appearing

1. Check Crowdin project dashboard — are source files synced?
2. Check if translated files exist at `content/docs/{locale}/`
3. Verify locale is listed in `src/lib/i18n.ts` and `src/lib/source.ts`

### Cloudflare deploy fails

1. Check GitHub Actions → Deploy workflow → logs
2. Verify `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID` secrets are set
3. Verify Cloudflare Pages project name matches `jamjet-docs`

### Local dev issues

```bash
rm -rf .next .source node_modules
npm install
npm run dev
```

### Adding Fumadocs MDX components

Fumadocs supports callouts, tabs, and other components. Import them in the page component or use markdown syntax:

```mdx
> **tip:** This is a tip callout

> **note:** This is a note
```
